# 下一个更大的元素1

力扣496

## 初步思路
下一个更大的元素就是和刚才的每日温度一样，从x往后数第一个比自己大的元素

这个多了一个功能，nums1和nums2，1是2的子集

我们先查nums1的元素在nums2的哪里，，然后定位到nums2后，再去像我们刚才每日温度的那个构建一个最小栈


但是这样有点很暴力的解了

回想一下接雨水，我们需要找到i到n的最大是怎么做的，

如果要找到后面的最大，那么需要我们从后往前遍历，，但是之前是找的最大，我们现在想要找到比i位置的第一个大的，或者说，我现在需要一个东西，去存储这个信息

## 答案思路

其实这个是在动态去做的，、

首先是，当我们从后往前遍历的时候，比如遍历到num，拿num往里面比，遇到比自己大的就让它滚，自己进去，如果没有人比自己大，就说明右边无比自己大的，直接冲进就行了，如果比完后发现栈还剩一个比自己大的，那么这个不就恰好是“最右边比自己大的嘛，

自然想到一个问题，如果让别人滚了，别人比自己小，那么继续往前遍历的时候，遇到一个比那个之前滚的小的呢

其实不会，因为是从后往前遍历的，而且这是动态的，对于一个num来说，栈顶那个东西一定是自己下一位的，按照顺序排好的，

那么这样的话我们就可以统计nums2中每一个位置的最右边第一个比自己大的是谁了，

因为nums1是nums2的子集，题目中说是不重复的，那么直接弄个hash，key是nums2的所有数字，value是最右边第一个比自己大的，最后查hash就行



```python 

class Solution:
    def nextGreaterElement(self, nums1: List[int], nums2: List[int]) -> List[int]:
        res = {}
        stack = []
        for i in range(len(nums2)-1,-1,-1):#倒着遍历
            while stack and nums2[i] > stack[-1]:#开始比较
                stack.pop()
            #比完后，如果stack空了，说明当前的num就是最大的，右边没有比自己大的了。hash里面就是-1
            #如果没有空，直接让那个栈顶当当前自己的右边的最大的
            res[nums2[i]] = stack[-1] if stack else -1
            #记得把nums加进去
            stack.append(nums2[i])
        #最后从hash里面取
        return [res[num] for num in nums1]

```
