# 并查集

emmm，并查集和栈、队列一样也是一种数据结构

并查集是干嘛的捏

举个例子，

A在东兴帮，B也在东兴帮，他俩都是C的小弟

有一天打架，A和B两个小喽啰过来，作为党鞭的我，怎么判断出A和B是不是我们帮派的

此时并查集就可以用了

并查集就像刚才所说的，是一个数据结构，那么自然也可以抽象成类

有了类就一定有方法，并查集的方法一般有三个

find方法，find(x)即找到x是谁的老大，这里自带了路径压缩，待会细说，

union方法，union (x,y)合并两个帮派（就是直接把两个帮派的老大合并，即A帮派的老大直接认B帮派的老大当老大

is_connection 方法，判断两个人是不是一个帮派的（就是看是不是一个老大的就行）

所以并查集有没有发现，union方法和is_connection方法中，都有一个认老大的

所以关键在于find老大的方法，这里也存在着路径压缩

## 并查集类

```python 
class UnionFind:
    def __init__(self, size):
        """
        初始化并查集。
        size: 元素的总个数（假设元素编号从 0 到 size-1）
        """
        # 核心本质：这就是那个记录“谁是老大”的数组
        # 初始时，[0, 1, 2, ..., size-1]，每个人的老大都是自己
        self.parent = list(range(size))

    def find(self, x):
        """
        查找操作（Find）：找到 x 的最终老大。
        这里自带了“路径压缩”的魔法。
        """
        # 如果自己的老大不是自己，说明上面还有人
        if self.parent[x] != x:
            # 递归顺藤摸瓜找最终老大，并且【重点】把沿途找到的老大直接赋值给 x 的直接上级
            # 这样下次查 x 的时候，一步就能找到最终老大，这就是“路径压缩”
            self.parent[x] = self.find(self.parent[x])
        
        return self.parent[x]

    def union(self, x, y):
        """
        合并操作（Union）：让 x 所在的帮派和 y 所在的帮派合并。
        """
        # 先分别找到这两人的最终老大
        root_x = self.find(x)
        root_y = self.find(y)

        # 如果老大不是同一个人，说明是两个不同的帮派
        if root_x != root_y:
            # 合并逻辑极其简单粗暴：让其中一个老大，认另一个做老大
            self.parent[root_x] = root_y
            # 注意：这里也可以引入“按秩合并”，但通常只用“路径压缩”速度就已经足够快了

    def is_connected(self, x, y):
        """
        实用方法：判断 x 和 y 是不是自己人（是否连通）。
        """
        # 只要最终老大是同一个人，那就是一家人
        return self.find(x) == self.find(y)

```

一个很好的例子


```python 
# 1. 实例化一个包含 5 个人的并查集
uf = UnionFind(5)

# 此时大家互相都不认识
print(uf.is_connected(0, 2))  # 输出: False

# 2. 发生关系合并
uf.union(0, 1)  # 0 和 1 成为好友
uf.union(1, 2)  # 1 和 2 成为好友

# 3. 验证连通性
# 0 和 2 哪怕没有直接加好友，但通过 1 连接，他们也是一个圈子的了
print(uf.is_connected(0, 2))  # 输出: True
print(uf.is_connected(0, 3))  # 输出: False (3还是个孤家寡人)

# 查看底层的 parent 数组长什么样
print(uf.parent) 
# 输出结果大概率是类似 [2, 2, 2, 3, 4] 这样的结构（取决于具体的指向顺序）
# 这说明 0, 1, 2 的最终老大变成了同一个数字。

```

写一个厚米的解释把，我已经理解了

第一阶段：建群与认大哥（Union 的过程）

初始状态： 大家互不相识。

parent = [0, 1, 2, 3, 4]

uf.union(0, 1)： 0 认 1 做老大。

由于最初 find(0) 是 0，find(1) 是 1，所以代码执行 parent[0] = 1。

当前数组变为：parent = [1, 1, 2, 3, 4]

uf.union(1, 2)： 1 认 2 做老大。

此时 find(1) 是 1，find(2) 是 2，代码执行 parent[1] = 2。

当前数组变为：parent = [1, 2, 2, 3, 4]

注意此时的物理状态： 0 的直接上级是 1，1 的直接上级是 2，2 的上级是自己。

这是一条链：0 -> 1 -> 2

第二阶段：顺藤摸瓜与路径压缩（Find 的过程）

现在我们要执行 print(uf.is_connected(0, 2))，这会分别触发 find(0) 和 find(2)。find(2) 很简单，因为 parent[2] == 2，直接返回 2。

最精彩的在于 find(0) 的执行轨迹。

我们开始递归拆解：
➡️ 【递】的过程（一层层往上问）第 1 层调用：find(0)程序检查：parent[0] 的值是 1。1 != 0，说明上面还有人。

程序准备执行赋值：self.parent[0] = self.find(1)为了得到等号右边的结果，程序被挂起，进入下一层递归...

第 2 层调用：find(1)程序检查：parent[1] 的值是 2。2 != 1，说明上面还有人。

程序准备执行赋值：self.parent[1] = self.find(2)为了得到等号右边的结果，程序被挂起，进入下一层递归...

第 3 层调用：find(2)程序检查：parent[2] 的值是 2。2 == 2，说明到底了，自己就是最高老大。

触发边界条件！开始返回结果 2。

⬅️ 【归】的过程（带着结果往下传，顺手修改数组）

返回第 2 层：刚才等在这里的代码 self.parent[1] = self.find(2) 终于拿到了结果。变成 self.parent[1] = 2（其实原来就是 2，这里重新赋了一次值）。

第 2 层结束，向上返回最终老大 2。返回第 1 层

（高潮来了！）：刚才等在这里的代码 self.parent[0] = self.find(1) 拿到了从上面传下来的结果 2。

执行操作：self.parent[0] = 2这里就是路径压缩发生的一瞬间！ 0 号元素的直接上级，从原本的 1，被强行修改成了终极老大 2。

最终，第 1 层调用结束，返回 2。

总结：经历了什么变化？在 find(0) 执行完毕的瞬间，底层的数组发生了一次隐秘的突变：

执行前： parent = [1, 2, 2, 3, 4] （0 连着 1，1 连着 2）

执行后： parent = [2, 2, 2, 3, 4] （0 直接连着 2，1 也直接连着 2）

这就是为什么说递归在这里极其优雅：它利用了递归的“回溯（归）”阶段，把刚刚辛辛苦苦探出来的底（最高老大 2），顺手发给这条路上经过的所有节点（节点 0），让他们直接越级指向最高老大。下次再 find(0)，就是 $O(1)$ 的时间复杂度了！



回到最初的话题，如果A是东兴，B是洪门，怎么快速判断A和B是不是同一个帮派的呢，在建帮派的时候这俩人就不是一个老大，因此我们is_same后，表里存的就是这俩的老大，直接O1的复杂度查出来，非常的好用
