什么是累计数，先问一下厚米

假设你有一棵二叉搜索树（BST）。题目要求你把这棵树上的每一个节点的值，都更新为：它原来的值 + 树上所有比它大的节点的值的总和。

搜嘎，也就是说，我从最右边的子树开始，因为没有比它大的了，所以就是自己，它的父节点只有左子树比自己大，那么就加上，父节点的左孩子就是加上后的父节点再加自己

感觉就是一个后序遍历啊，是吧，

啊不对不对，这里其实是 右-中-左，这是什么遍历手法

嗷嗷嗷，这是中序遍历的逆序版本，那我其实也这么遍历就行

但是单纯的遍历好像是不太行，因为递归遍历的话，比如我到左节点，其实我是要加上这个父亲节点的值的，所以我需要一个东西来去记录这个父节点的值

此外，回想一下，这个里面是二叉搜索树，肯定要用性质，这里说了是中序的逆序版本，那么可以想象一下，我把这个树中序打出来，然后从后往前累加不就行了哈哈哈，跟我们做的那个众数差不多，都是树上的双指针

    def convertBST(self, root: Optional[TreeNode]) -> Optional[TreeNode]:
        self.pre = 0
        self.in_order(root)
        return root

    def in_order(self,node):
        if not node:
            return 
        self.in_order(node.right)
        if self.pre == 0:#如果是0，说明是刚开始到最右边
            self.pre = node.val
        else:#不是0，说明有了
            node.val += self.pre
            self.pre = node.val
        self.in_order(node.left)
