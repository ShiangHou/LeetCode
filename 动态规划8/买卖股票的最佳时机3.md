其中k是长度，从0到n天，然后i是0和1，第i天买还是不买，j是第几次，这样可以扩展到k次的情况

所以这样我们就可以表达出，第i天的第几次的买还是不买

定义三维数组 dp[i][k][s]，其物理含义为：在第 $i$ 天结束时，已经启动了 $k$ 次交易，且当前的持仓状态为 $s$ 所获得的最大利润。$i$ (天数)：取值范围 $[0, n-1]$。$k$ (交易次数)：取值范围 $[0, K]$。在本题中 $K=2$。核心约定：我们将“买入”动作视为一次交易的启动。 也就是说，只要执行了买入操作，$k$ 的值就必须加 1。卖出操作不增加 $k$。$s$ (持仓状态)：取值为 $0$（不持有股票）或 $1$（持有股票）。

即，只要是买了，次数就要加1

这么看来还是对的，以买来去链接

dp[i][k][0] 因为是k，所以说有两种，即昨天也没买，dp[i-1][k][0]

还有一种是，昨天买了，但是卖了，dp[i-1][k][1]+ prices[i]


dp[i][k][1] 即 如果昨天已经持有了股票，那么就是即成，dp[i-1][k][1]

如果昨天没有持有,那么昨天一定是k-1的状态，所以是  dp[i-1][k-1][0] -prices[i]

此外注意好边界条件
k = 0的时候，没有进行任何交易
dp[i][0][0] = 0,但是知道dp[i][0][1]是不合法的，所以弄成float（-inf)

i = 0的时候，
dp[0][k][0] = 0这个是都没买，都是0
dp[0][k][1] = -prices[0] 无论什么时候都是花钱，，因为第0天持有了


```python 
class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        k = 2
        n = len(prices)
        dp = [[[-float('inf')]*2 for _ in range(k+1)] for _ in range(n)]
        
        #初始化，k = 0的时候，状态是0的话，是0，是1的话，全部弄成inf
        for i in range(n):
            dp[i][0][0] = 0
            dp[i][0][1] = float('-inf')
            
        #i为0的时候，0状态就是0，然后1状态就是-prices【0】
        for i in range(k+1):
            dp[0][i][0] = 0
            dp[0][i][1] = -prices[0]
        #开始填表
        for i in range(1,n):
            for j in range(1,k+1):
                dp[i][j][0] = max(dp[i-1][j][0],dp[i-1][j][1]+prices[i])
                dp[i][j][1] = max(dp[i-1][j][1],dp[i-1][j-1][0]-prices[i])
        return dp[n-1][k][0]
```
