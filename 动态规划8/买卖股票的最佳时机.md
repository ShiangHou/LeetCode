## 初步思路

第一个遍历一遍，如果后面的比前面的大，直接算差，累加就行

嗷嗷不对，这个是只能买一次股票，问这一次能够获取的最大的价值

那么就是，我找到这个数组中的最小的是i，然后找到这个i后面比他大的最大的就行

换句话说，我在遍历的时候，我取最小值作为cost，然后取当前的减去最小的作为利润，找利润最大的就行

因为是不断往前推的，每次的利润都是未来的价格，所以是可以的

```python 
class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        cost,Profit = float('inf'),0
        for i in range(len(prices)):
            cost = min(cost,prices[i])
            Profit = max(Profit,prices[i]-cost)
        return Profit  
```

## dp

动态规划的做法是，所谓的持仓

因为我们有天数，dp[i][j] 其中i是天数，j是0和1，即在第i天我手里还有没有股票 假设0是有票，1是没票

然后这个dp里面的值是得到的利润

那么思考一下初始条件，dp[0][0]就是第0天没持有，那么利润就是0,那么dp[0][1]就是-x，即第0天持有，但是没有利润，所以是负的

递推公式，如果dp[i][0] ,即第i天没有持有，那么利润其实就是等于上一天的，


即max(dp[i-1][0],dp[i-1][1])

不对，如果dp[i-1][1]，但第i天是没持有，说明第i天卖了呀，那么就是

dp[i-1][1]+price[i]

所以是max(dp[i-1][0],dp[i-1][1]+price[i])


dp[i][1]，说明第i天持有了，那么有可能是i-1天也持有了，所以dp[i-1][1]

也有可能i-1天没买，是i天买的，那就是dp[i-1][0]-price[i]
所以是max(dp[i-1][1],dp[i-1][0]-price[i])

不对，这里dp[i-1][0]-price[i] 不能这么写，如果今天买了，那么利润就是直接-price[i]就行，因为之前不能买，只能今天买

```python 
class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        n = len(prices)
        dp = [[0]*2 for _ in range(n+1)]#0是没持有，1是持有
        dp[0][1] -=prices[0] 
        for i in range(1,n):
            dp[i][0] = max(dp[i-1][0],dp[i-1][1]+prices[i])#前一天本来就没持有，或者前一天持有了，今天卖了
            dp[i][1] = max(dp[i-1][1],-prices[i])#前一天本来就持有了，或者前一天没持有，今天买了
        return dp[n-1][0]#直接返回最后一天没持有的状态



```
