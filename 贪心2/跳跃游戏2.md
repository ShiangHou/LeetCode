## 初步思路

题目跟跳跃游戏是差不多的，那么问题是，题目要求在保证数组一定是能跳到最后的情况下，问我们最少能跳多少次

初步的想法是，只有当我的最大覆盖范围变大了，那么此时我跳是有进步的，那么我才会跳到这里

左神的思路是

比如数组是 5 2 1 8 0 4

当我站在0位置的时候，我需要考虑走nums[0]的哪一步能抵达最远的下标

比如这里，在0位置，nums[0]是5 ，那么
我走0步，就是0

我走1步，走到2 ，那么可以最远到达3（因为1+nums[1]

我走2步，走到1 ，那么最远可以抵达 2+nums[2]

以此类推

发现其实最远可以抵达3+nums[3] 即11 什么时候覆盖了就停止


## 最终思路

好理解，但是代码思路很好,就是在我们的饿跳跃游戏加了一个，如果当前的cur没有覆盖到i，就要走一步

这个




```python 
    def jump(self, nums: List[int]) -> int:
        cur = 0 # 当前步数，最右到哪
        _next = 0 #再走一步，能到哪
        ans = 0
        i = 0
        while i < len(nums):
            #来到了i下标
            #如果当前的步数大的覆盖值包含了i，那么说明不用付出额外的步
            #如果没覆盖，需要步数+1
            if cur < i:
                ans += 1#需要再走一步
                cur = _next
            _next = max(_next,i+nums[i])
            i+= 1
        return ans
```


这个next其实本质上就是能往外扩展的最大距离

代码意思就是，如果我当前走的这个位置超出了我原来能覆盖的最大范围，我就需要走一步（不管是在哪里走的，总之是要走），如果没超过，说明在之前

换句话说，还是那个例子，我在0位置是5

首先我的max是正常统计的

然后我的cur是滞后的，
如果我发现走到了i=6，但是我滞后的max没有覆盖到6，说明什么

说明在0-5这个范围内能有让我到达6的点，虽然不知道是那一个，但是总是用的，所以此时我要更新这个滞后的信息，cur = next，（当然前提是总能抵达最后，如进阶一点这里可以加条件判断，cur和next关系判断还能不能往下走）
