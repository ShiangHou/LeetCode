## 初步思路

给了一个数组，然后，需要找到数组中递增的子序列

一个很暴力的想法就是说，我定义dp[i]是以i结尾的，位置从0-i的最长递增子序列

那么只需要一点一点填dp就行，从i遍历到n，对于每一个i，刷dp，对于0-i位置的每一个j，如果这个数是小于nums[i]的，就取

即递推是$$dp[i] = \max_{0 \le j < i, nums[j] < nums[i]} (dp[j] + 1, dp[i])$$

但是有问题，这个只是再找比nums[i]小的，在i位置之前的数量，但是怎么保证i位置之前都是“递增”的呢

比如 5 8 2 3 4 7，看到最后一个7的时候，去遍历前面所有的比7小的，首先是5，ma

事实上，在遍历前面的时候，需要去判断一下，max(dp[i]，dp[j]+1)谁大，

换句话说，如果中间是断了的（比如上面的 5 8 2 的2），那么在看2 的时候，它前面是只记录了比2 小的，

再比如，5 8 2 10 4 7


5 位置是1 ，8位置是2 2位置是1 

到了10位置，因为我们的2位置是1 所以说这种小的截断没有 8 那个位置的大，所以10位置会继承最大的，即8位置的，从而避免中途出现的小的2 导致的截断

最后我们去dp所有的最大的那个就行

最终思路就是，初始化dp为1，然后
遍历i从0到n，
遍历j从0到i，如果nums[i] > nums[j]，那么就取 dp[i] = max(dp[i],dp[j]+1)

return maxdp

```python 
class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        dp = [1]*(len(nums))
        for i in range(len(nums)):
            for j in range(i):
                if nums[i] > nums[j]:
                    dp[i] = max(dp[i],dp[j]+1)
        return max(dp)
```

## 进阶

可以发现，这个东西每次需要重复比较，

即每计算一个i的时候，需要和前面所有的都比较，白白浪费，因为对于历史信息没有进行有效的组织，

所以现在需要组织，我们定义一个tail，假设tail数组的长度为k，那么就是

所有长度为 k+1 的合法递增子序列中，尾部数值最小的那一个状态。

比如 现在是 2 3 8 和 2 3 5，如果来了一个6的话，可以接在5的后面，但是不能接在8 的后面

所以，这里就是，如果来了一个数字，我把它和tail[-1]比较，如果比它大，那么直接tail.append,如果小的话，我需要找到tail中第一个比自己大的，然后替换掉就行，找的可以用二分，最后遍历玩数组，tail的长度就是最后的长度

所以思路就是，遍历数组，构建tail，如果说比-1大，就放入，小的话就二分，所以需要写一个主进程，一个二分


```python 
class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        tail = [nums[0]]
        for i in range(1,len(nums)):
            #如果比尾巴还大，直接加
            if nums[i] > tail[-1]:
                tail.append(nums[i])
            else:
                max_left = self.bisearch(tail,nums[i])
                tail[max_left] = nums[i]
        return len(tail)


    
    def bisearch(self,nums:list, x:int) -> int:#找到数组中第一个比x大的，替换掉,返回元素的下标
        l,r = 0, len(nums)-1
        while l < r:
            mid = (l+r) //2
            if nums[mid] < x:#说明比x大的在右边
                l = mid+1
            else:#不能减 1，因为 mid 本身可能就是我们要找的“第一个大于等于 x”的值
                r = mid
        return l

```
