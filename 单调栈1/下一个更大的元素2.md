## 初步思路

在下一个更大元素1的时候，我们学会了动态的统计右边第一个比自己大的

这题也是要求这个，但是他说数组是循环的，就是n后面是-1

emmmm那这样从后往前遍历肯定就不行了

从前往后遍历呢，好像是可以的，

但依旧有一个问题是

比如 4 3 2 5 1 0

5来的时候把 4 3 2都寄走了，到1的时候后面没有了，或者说后面还有一个，但是其实是在前面

换句话说，走完一遍后，我的栈里面肯定还剩，比如这里还剩两个，这两个是按照降序排列的，比如1 0

那么我可以在扫一遍数组，从头开始，并且这里我不做append，如果大的话直接让里面的弹出就行了，如果最后，如果发现stack弹完了，说明答案搜集完了，返回，如果第二遍过了一遍发现stack还没完，说明没有人比他大，直接弄成-1返回

```python 
class Solution:
    def nextGreaterElements(self, nums: List[int]) -> List[int]:
        ans = [-1]*len(nums)
        stack = []
        for i in range(len(nums)):
            while stack and nums[i] > nums[stack[-1]]:#说明栈里面的那个stack[-1]找到了比自己大的
                ans[stack[-1]] = nums[i]#记录一下答案
                stack.pop()#弹出
            #最后把自己加上
            stack.append(i)
        #最后至少还剩一个，再过一遍
        for num in nums:
            while stack and num>nums[stack[-1]]:
                ans[stack[-1]] = num
                stack.pop()
            #如果过到一半发现已经空了，提前返回
            if not stack:
                return ans
        #最后直接返回，因为即使stack没空，那说明有的本来就没自己大，默认的就是-1，
        return ans 
```
