# 加油站

力扣134

## 初始思路

这题说的是，我有两个数组，一个数组是cost，一个数组是gas，如果我从某一个位置i出发，前往下一个位置需要消耗cost i,同时当我到达位置i的时候，我就会获得gasi，问能不能绕一圈（即遍历完所有的i位置）出发的时候没有油

这题初步是思路是这样的，首先，我的出发点一定是 gas 要大于等于cost的，然后我从这个出发点出发，开始往后走，如果发现这个油箱里的油小于，那么直接返回，

当遍历到头后，看一下0位置有没有被遍历到，如果没有，那么就转到0位置就行

所以说用一个set来去存下标的位置来去判断有没有到达过该位置，（即到一个位置直接在集合中删除掉该位置就行，最后看一下集合是不是空的就行）

```python 
class Solution:
    def canCompleteCircuit(self, gas: List[int], cost: List[int]) -> int:
        ans = -1
        for i in range(len(gas)):
            if self.f(i,gas,cost,len(gas)) == i:
                return i
        return ans 

    def f(self,start,gas,cost,n):#从start开始可以不可以走完
        if gas[start] < cost[start]:#如果起步都不够，肯定不行
            return -1
        rout = set(i for i in range(n))
        oil = 0
        j = start
        while rout:#当还可以走的时候
            #先加一下当前位置的油
            oil += gas[j]
            if cost[j] > oil:#不够往下走了
                return -1
            else:#还能往下走
                oil -=cost[j] #先扣油
                if j == n-1:#如果走到了最后,说明下一步是要到0位置
                    j = 0
                else:#正常位置，往下走
                    j += 1
            rout.remove(j)
            if j == start:#如果又走回来了，说明走一圈了，成功了
                return start
```

这种写法超时了，有点暴力，即遍历所有符合条件的点，然后去跑一遍

当然复杂度是On2


## 答案思路
其实有小巧思

如果说gas的和大于cost的和，那么一定有解

然后我们从0开始找解，模拟那个过程，如果遇到cost>gas,那么我们就立即从i+1的位置，清空所有的gas重新开始

最后复杂度是On


```python 
    def canCompleteCircuit(self, gas: List[int], cost: List[int]) -> int:
        #如果不大于，就一定跑不完
        if sum(cost) > sum(gas):
            return -1
        oil = 0
        start = 0
        j = start
        while j < len(gas):
            #先加油
            oil += gas[j]

            if cost[j] > oil:#如果发现不行，清空
                start = j+1
                oil = 0
            else:
                oil -= cost[j]
            j += 1
        return start


```
