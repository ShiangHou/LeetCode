dpn就行

对于第i个，需要看一下

如果i用了，那么dp[i-1]一定没有用，并且是和dpi相差了nums【i】的，即 dp[i] = dp[i-1]+nums[i]

如果i没用，那么dp[i-1]其实可用可不用，因为这个要看i-2有没有用啊如果i-2还没用，那么i-1就一定要用，不然不用白不用。i-2如果用了的话，i-1就不能用

如果i-2用了，那么就是 dp[i-1] = dp[i-2],因为i-1不能用，而且现在是i也没有用，那么就是有dp[i] = dp[i-1] = dp[i-2]

如果i-2没有用，i也没有用，那么dp[i-1]就一定要用，所以dp[i-1] = dp[i-2]+nums[i-1]=dp[i]

看了一下，总之这两种情况都是dp[i] = dp[i-1]

所以递推是 dp[i] = max(dp[i-1],dp[i-1]+nums[i])

啊不对，这样的话dp[i-1]+nums[i]一定会大于的，所以要用dp[i-2]+nums[i-1]这个吧

dp[i] = max(dp[i-1],dp[i-1]+nums[i]，dp[i-2]+nums[i-1])

感觉还是不太对啊

算了从暴力递归来去分析一下吧

对于一个暴力递归
```python 

def f(i,sum):
    if i>= len(nums):
        return 
    #用了i，直接跳到i+2
    sum += nums[i]
    f(i+2,sum)
    #没用，记得先减回来
    sum -= nums[i]

    f(i+1,sum)
```

哎哎哎，暴力递归这里，是直接跳到了i+2，意思就是说，如果当前用了，那么我直接跳到i-2就行了，搜噶

所以就是，用了i，dp[i] = dp[i-2]+nums[i]

如果没用,暴力递归这里直接传到了i+2，那么同理，我直接依赖i-2就行
dp[i] = dp[i-1]


```python 
class Solution:
    def rob(self, nums: List[int]) -> int:
        n = len(nums)
        dp = [0]*(len(nums)+1)
        dp[1] = nums[0]

        for i in range(2,n+1):
            dp[i] = max(dp[i-1],dp[i-2]+nums[i-1])
        return dp[n]
```

注意，这里和我们之前讲的一样，要加nums[i-1]，然后前面0是0，1是原来nums的第一个，对其就行
