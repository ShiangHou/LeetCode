## 初步思路

这道题说，给了我们一个矩阵，矩阵代表了高度，，然后左上边代表了一个边界，右下两个边代表着另一个边界

矩阵里面的元素是高度，水可以从高处往低处流，

问有哪些矩阵中的元素可以流向两个边界，

这个题目其实也是深搜或者广搜，反正就是一直找，

比如我们可以用递归，首先遍历所有元素，然后前后左右四个方向 取or，然后递归的边界是如果遇到高的返回false，如果遇到边界返回True

但是这样只能判断出，从i点出发能不能抵达边界，怎么去判断能不能同时抵达两个边界，

那就不用True或者false，用三个变量，假设是0 1 2，0代表着没有遇到边界，1代表着遇到了左上边界，2代表着遇到了右下边界

那么在递归的时候怎么接住这个返回呢，就不能只用or了，需要对上下左右四个方向进行判断，判断他们返回的是什么

那么其实情况有很多，首先是四个方向全都是0，说明就没有路，此时返回0就行了

如果四个方向就只有1，其他全是0，那么说明这个格子就只能抵达左边界，往上返回1就行

同理，如果只有2和0，往上返回2

如果1和2都有，说明这个格子符合条件，此时我们往上返回3，表示都可以

那么同理，如果自己的小弟也返回了3，那么自己也往上返回3

嘶，那么我们可以用一个记忆话搜索来去完成，因为是递归，所以每个格子都有自己的状态，即0 1 2 3的状态，然后可以用一个visited来去存，遍历的时候如果发现没有遍历到，那么就启动递归，如果在递归的过程中发现之前已经计算过了，直接返回就行

最后遍历这个visited就行，如果是3，就返回graph里买呢元素，

那么其实递归的时候不断的去填这个visited表就行

那么其实可以初始化啊，就是左上的边弄成1，右下的边弄成2，两个顶点弄成3

```python 
n,m = map(int,input().split())

#建图
graph = []
for i in range(n):
    graph.append(list(map(int,input().split())))

#方向,这里的顺序分别是右，左，上，下
direction = [[1,0],[-1,0],[0,1],[0,-1]]

#这里是记忆化搜索，用来存储状态 
#0是哪都到不了，1是可以仅到达左上，2是可以仅到达右下，3是两遍都可以到达
visited = [[-1]*m for i in range(n)]

#初始化，左上弄成1，右下弄成2，交汇处弄成3

#第一行是1，最后一行是2
for i in range(n):
    visited[i][0] = 1
    visited[i][m-1] = 2

#第一列是1，最后一列是2
for j in range(m):
    visited[0][j] = 1
    visited[n-1][j] = 2

#两个角是3
visited[0][m-1] = 3
visited[n-1][0] = 3


#深度遍历，去填visited表就行
def dfs(graph,i,j,visited):
    #如果是visted已经遍历到了，直接返回这个就行
    if visited[i][j] != -1:
        return visited[i][j]
    
    else:
        #上下左右开始遍历，因为这里的边界都初始化了，不会溢出，溢出的边界都已经初始化了，走不到这


        #这里准备一个数组存放这四种状态,这里聪明一点，拿下标表示状态，值表示出现了几次
        temp_state = [0,0,0,0]
        for x,y in direction:
            next_i,next_j = i+x,j+y
            #因为这里的边界都初始化了，不会溢出，溢出的边界都已经初始化了，走不到这,
            #这里只需要考虑四个方向哪个是低于自己的，如果是低于自己，才会冲进去递归，如果比自己高，直接0就行
            if graph[next_i][next_j] > graph[i][j]:
                temp_state[0] +=1
            else:
                #递归，返回这四个方向的四个状态,填入对应状态下标的数量
                temp_state[dfs(graph,next_i,next_j,visited)] +=1



        #然后看一下这四种状态的情况
        #都是0就返回0，只有一个1就返回1，只有2就返回2，有一个3直接返回3，即有1也有2就返回3
        if temp_state[3] != 0:#出现3了
            visited[i][j] = 3
        elif temp_state[1] !=0 and temp_state[2]!=0:#有1也有2
            visited[i][j] = 3
        elif temp_state[1] != 0 and temp_state[2]==0:
            visited[i][j] = 1
        elif temp_state[1] == 0 and temp_state[2]!=0:
            visited[i][j] = 2
        else:
            visited[i][j] = 0
    
    #最后return当前的状态
    return visited[i][j]

def main():
    #主函数
    #先执行一下,边执行边返回
    for i in range(n):
        for j in range(m):
            if dfs(graph,i,j,visited) == 3:
                print(i,j)
                
if __name__ == "__main__":
    main()

```

这么写感觉有问题，就是如果两个高度是一样的，然后假设都是10，然后四个方向遍历完，10问10，然后另一个方向也是10问10，这样会陷入死循环

需要修改几点

1:不要是边界，因为边上也有可能是往左右流，所以初始化只初始两个点就行，剩下的放到递归里面判断

2:先把自己标为-2，如果递归又递归回自己，直接忽略掉

## 逆向思路

dubug弄不出来了，逆向思维，就是从边界开始往里面，左边界开始往上走，右边界开始往上走，然后找两个交集就行

也就是说，我从左上的边界开始，dfs，找到所有能够升到的地方，然后从右下开始，找到能够升到的地方，最后看两个的都是True就行


```python 
n,m = map(int,input().split())

graph = []
for i in range(n):
    graph.append(list(map(int,input().split())))

direction = [[1,0],[-1,0],[0,1],[0,-1]]

left = [[False]*m for i in range(n)]#左上的能够到达的地方

right = [[False]*m for i in range(n)]#右下的能够到达的地方



def dfs(graph,i,j,visited):
    visited[i][j] = True

    for x,y in direction:
        next_i,next_j = i+x,j+y
        if 0<= next_i<n and 0<=next_j<m:#如果还在范围内
            if visited[next_i][next_j] == False and graph[i][j]<= graph[next_i][next_j]:
                dfs(graph,next_i,next_j,visited)
    
def main():
    #从左边界和上边界开始
    for i in range(n):
        dfs(graph,i,0,left)
        dfs(graph,i,m-1,right)
    for j in range(m):
        dfs(graph,0,j,left)
        dfs(graph,n-1,j,right)

    #找到交集
    for i in range(n):
        for j in range(m):
            if left[i][j] == True and right[i][j] == True:
                print(i,j)

if __name__ == "__main__":
    main()
```
