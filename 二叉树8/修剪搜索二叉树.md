## 思路

看了一眼题目，very的复杂啊，题目说，给一个范围low-max，要求对于整棵二叉树不在这个范围里面的全部都斩杀，斩杀完记得保存树的原来面貌

昨天其实做过删除操作了，这里感觉还是有点意思

首先对于一个搜索二叉树，如果val值小于了min，那么它的所有的left都应该被斩杀，

提问：那我们需要考虑右边嘛

答：不需要，因为我们是一路上从root来判断的，如果root合法，根据二叉搜索树的规定，左节点的右子树都不能大于root，因此当左节点不合法时，左节点的右子树应该

啊不对不对，是需要的，因为左节点的右子树虽然不会大于max，但不一定大于min啊，还是要递归的去判断，但是左节点的左边肯定是要全部斩杀


梳理一下思路，对于一个node，如果小于min，那么左边全都弄成空，然后递归调用右边

如果大于max，那么右边全都是空，递归调用左边

如果正常，就递归左右


```python 
    def trimBST(self, root: Optional[TreeNode], low: int, high: int) -> Optional[TreeNode]:
        if not root:
            return root
        if root.val < low:#小于low的话不用管左边，只返回右边的递归结果就行
            return self.trimBST(root.right,low,high)
        elif root.val > high:#同理
            return self.trimBST(root.left,low,high)
        else:#该节点正常，都往下走
            root.left = self.trimBST(root.left,low,high)
            root.right = self.trimBST(root.right,low,high)
        return root
```
