## 思路

经过厚米Gemini的提示，豁然开朗

并查集擅长的是合并，而不是拆，如果图已经构建好的话，所有人都指向自己的老大，这其实一点都拆不出来

因此不要想着拆，而是边建立边判断，如果发现建立的这条边有问题，立即输出就行

什么叫建的边有问题

比如说我建边的时候，在建边前，看一下两个x和y的老大

如果x和y已经是同一个老大了！！（即find(x)和find(y)，让路径压缩一下）

那说明这条边就没有用啊，直接输出这个答案

而且，这样还可以保证顺序，看例子说的，如果前面所有的边都是合法的，最后一条加了反而不合法，直接就会在建最后一条边的时候斩杀


```python 
#先写一个并查集的类
class UnionSet:
    def __init__(self,n):
        self.father = [i for i in range(n)]
    
    def find(self,x):
        if self.father[x] == x:
            return x
        else:
            self.father[x] = self.find(self.father[x])
            return self.father[x]
    

    def union(self,x,y):
        father_x = self.find(x)
        father_y = self.find(y)
        if father_x != father_y:
            self.father[father_x] = father_y

    def is_union(self,x,y):
        return self.find(x) == self.find(y)

def main():
    n = int(input())
    parents = UnionSet(n+1)
    for _ in range(n):
        x,y = map(int,input().split())
        #开始前先判断俩是不是一个老大
        if parents.is_union(x,y):
            #如果是的话，直接存答案
            ans_x,ans_y = x,y
        else:#不是的话，正常建
            parents.union(x,y)
    return ans_x,ans_y

if __name__ == "__main__":
    ans_x,ans_y = main()
    print(ans_x,ans_y)

```
