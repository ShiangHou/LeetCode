## 初步思路

首先，题目说给我们一个数组，这个数组是，从下标i往上爬，的成本是costi

那么依旧是动态规划啊，只不过在跑楼梯的过程中，原来是i-1  和 i-2的和，现在是这两个的cost的最小值就行了

其实好像不是的，看题目的意思，位于i处，如果花费了i，那么我可以往上走1步或者两步

也就是说，当前这一步的花费，应该是之前的最低花费，加上自己的花费，也就是cost i 加上dp i-1和dp i-2取小的就行

嗷嗷，其实是，比较cost i-1和cost i-2 再加上各自的dpi-1和各自的i-2就行

也就是说，dp是到达这个地方的最小（不算脚底踩着的）

因此，初始化的时候是0，然后因为最后要计算到n位置，也就是最后一个数组的下一个

```python 
    def minCostClimbingStairs(self, cost: List[int]) -> int:
        n = len(cost)
        dp = [0]*(n+1) #这个意思是到达第i个的最小成本是多少
        dp[0] = 0
        dp[1] = 0
        for i in range(2,n+1):
            dp[i] = min(cost[i-1]+dp[i-1],cost[i-2]+dp[i-2])
        return dp[n]

```
