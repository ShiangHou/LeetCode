依旧是先用暴力递归去做

class Solution:
    def canPartition(self, nums: List[int]) -> bool:
        self.flag = False
        if sum(nums)%2 != 0:#如果是基的，就一定不行
            return False

        self.f(0,nums,0)

    def f(self,i,nums,temp_sum):
        if temp_sum == self.target:

            return

            return

首先我的递归有两个变量，一个是i，即当前遍历到的下标，一个是tem_sum,是当前的和

之前我们写的没有返回值啊，用的是全局变量，所以应该存一个，即i下标，然后tem_sum的情况下是True还是False

```python 
class Solution:
    def canPartition(self, nums: List[int]) -> bool:
        if sum(nums)%2 != 0:#如果是基的，就一定不行
            return False
        self.target = sum(nums) //2
        return self.f(0,nums,0,{})

    def f(self,i,nums,temp_sum,dp):#这个是要返回True和False
        if (i,temp_sum) in dp:
            return dp[(i,temp_sum)]

        if temp_sum == self.target:
            dp[(i,temp_sum)] = True
            return dp[(i,temp_sum)]
        if temp_sum > self.target or i >=len(nums):
            dp[(i,temp_sum)] = False
            return dp[(i,temp_sum)]
        #要i和不要i的合并，然后取or，拿到True和False
        dp[(i,temp_sum)] =  self.f(i+1,nums,temp_sum+nums[i],dp) or self.f(i+1,nums,temp_sum,dp) 
        return dp[(i,temp_sum)]
```

这样就可以过了

这里的意思是

从数组的索引 i 开始向后遍历，在当前已经累加的数值为 temp_sum 的前提下，最终能否凑出目标和 target

所以最后是从0 0 开始


## 动态规划

在递归的时候，我们用了两个变量，一个是i，一个是target

那么我们可以写一个dp[i][j],其中i是0-n ,j是0-target

然后这个表里面填True和False了，就像我们刚才写的那个dict一样，

那么我们的递推公式就是刚才的递归，即dp[i][j] 应该是 dp[i+1][j]

等一下，思考一下，我们的定义是什么

j的范围是target到0，由于这里需要从0往后一点点写，那么j应该是，前i个组合能不能组成j？

是的，

那么如果用了当下的，就要j去减就行，即递推公式是

dp[i][j] = dp[i-1][j] or dp[i-1][j-nums[i]]

此外我们需要判断一下是不是j是大于nums[j]，如果不是的话，那么我们其实只能选择不放i了，即dp[i][j] = dp[i+1][j]

此外，初始化，dp[0][j],用0个元素能不呢做到，这个是全都是0

dp[i][0] 能不组成0，当然是可以的，因为全都是不放，所以都是True


```python 
class Solution:
    def canPartition(self, nums: List[int]) -> bool:
        n = len(nums)
        if sum(nums)%2 != 0:
            return False
        target = sum(nums) // 2
        dp = [[False]*(target+1) for _ in range(len(nums)+1)]

        for i in range(n+1):
            dp[i][0] = True
        
        for i in range(1,len(nums)+1):
            for j in range(1,target+1):
                if j >= nums[i-1]:
                    dp[i][j] = dp[i-1][j-nums[i-1]] or dp[i-1][j]
                else:
                    dp[i][j] = dp[i-1][j]
        return dp[n][target]
```

这里有几个细节，首先是，dp[i][j] = dp[i-1][j-nums[i-1]] or dp[i-1][j]

这里为什么是i-1呢，即如果取了i，不应该是j-nums[i]嘛？

其实是对的，我们本质上还是j-nums[i]，但这里的i-1是工程代码上的问题，即

为了规避 $i-1$ 在 $i=0$ 时可能引发的数组越界问题，部分算法实现会将 $dp$ 数组的行数设定为 $n+1$（索引范围从 $0$ 到 $n$）。
此时，$dp[i][j]$ 的定义变更为：仅考虑 nums 数组中的前 $i$ 个元素，所以所以“前 $i$ 个元素”中的最后一个（即第 $i$ 个元素），其在 nums 数组中的物理下标为 $i-1$


比如下标3，i是3，但是我们考虑前3个元素，其实是2个，因为最开始的有个0哈哈哈

## 一维

注意到，i每次都是从i-1来的，即从填表的角度来看，i每次都只看上一行
所以对于i来说，我们就一值让i往后走就行了

然后对于j的话需要去做，注意到我们的j是一只在减小的，即开始是target，然后遍历的时候会减去

所以整理成1维就是


```python
class Solution:
    def canPartition(self, nums: List[int]) -> bool:
        if sum(nums)%2 != 0:
            return False
        target = sum(nums) // 2
        dp = [False]*(target+1) 

        
        dp[0] = True
        
        for num in nums:
            for j in range(target,num-1,-1):#这里要倒着，因为是一值在减去，
                dp[j] = dp[j] or dp[j-num]#这里也不用判断了，如果上面那个循环是到1的，需要判断，上面到num，就直接截断就行
        return dp[target]


```
